แต่ละไฟล์คืออะไรบ้าง
1.โฟลเดอร์ที่เก็บ “ของทั้งหมดที่โปรเจคต้องใช้” เช่น ไลบรารี React, TypeScript, Vite
– npm จะสร้างมันให้อัตโนมัติ
แก้ไขได้ไหม: ❌ อย่าแตะ!
ถ้าแก้จะเกิดอะไรขึ้น:
– โปรเจคพัง
– dependency หาย
– error บานปลาย

2. public/
คืออะไร:
– ไฟล์สาธารณะ เช่น รูป, ไอคอน, assets ที่ต้องการให้เข้าถึงโดยตรง
– ของด้านในจะถูกคัดลอกไป build สุดท้ายแบบตรงๆ
แก้ไขได้ไหม: ✔️ ได้
แก้อะไรได้:
– รูปภาพเกม
– เสียง
– ไอคอน favicon
ถ้าแก้:
– เว็บโหลด resource เปลี่ยนตามนั้นทันที

3. src/ (สำคัญที่สุด)
คือโฟลเดอร์ “ทั้งหมดที่เป็นตัวเกม” ของเรา
ทุก UI, logic, component อยู่ในนี้ค่ะ
3.1. src/assets/
– ใช้เก็บภาพ โลโก้ หรือไฟล์เฉพาะของเกม
– มักถูก import ผ่าน React/TS
แก้ได้ไหม: ✔️ ได้
ถ้าแก้: ไม่มีผลเสีย ใส่ resource ของเกมได้ตามใจ
